## 各脚本段审核

### 段一：fetch 拦截 + console 抑制（行 327-373） ✅

- `console.warn`/`console.error` 覆写，过滤 `hit_areas` / `D_REF` / `_$NP` 关键词
- `model_list.json` 请求拦截，返回三人列表
- `sagiri/index.json` 请求拦截，重定向到本地 `/js/model/sagiri/index.json`
- `sagiri/voice/*.wav` 请求拦截，重定向 CDN 路径到本地路径
- **结论：逻辑正确，无 bug**

**小建议**：`Array.prototype.join.call(arguments, ' ')` 在 console 传入对象参数时会丢失信息，可改进但不影响功能。

### 段二：Audio 音量 + 冷却 + 模型隔离（行 376-445） ✅

- `window.__lv2AudioHooked` 防 PJAX 重复覆写
- `modelOf(url)` 根据 URL 中的模型名判断归属
- `stopOthers(model)` 切换模型时暂停并重置上一个模型的 Audio
- `play()` 覆写实现 7 秒冷却（每个模型独立计时）
- `ended` / `pause` 事件清理 `liveAudios` 跟踪数组
- **结论：语音串扰、重复触发、7s 冷却均已解决**

**确认**：`modelOf` 用 `/umaru/` `/sagiri/` `/chino/` 匹配，CDN 和本地路径均兼容。

### 段三：错误抑制 + initWidget（行 448-477） ⚠️

- `window.addEventListener('error', ...)` capture 阶段拦截 Cubism2 运行时异常
- 过滤条件：`_$NP` / `D_REF` / `null` + 文件名 `live2d.min.js`
- `initWidget()` 加载看板娘，`cdnPath` 指向 yzs CDN
- **结论：功能有效，但有一个优先级隐患**

**运算符优先级问题**（行 452）：
```javascript
m.indexOf('_$NP') !== -1 || m.indexOf('D_REF') !== -1 
    || m.indexOf('null') !== -1 && e.filename && e.filename.indexOf('live2d.min.js') !== -1
```
`&&` 优先级高于 `||`，当前等价于：
```
(m 包含 '_$NP') → 拦截
(m 包含 'D_REF') → 拦截  
(m 包含 'null' 且文件名是 live2d.min.js) → 拦截
```
**实际上能拦住 hitTest 报错**（消息同时含 `null` 和 `hitTest`），但建议加括号或直接加上 `hitTest` 关键词。

### 段四：鼠标拖尾（行 479-483） ⚠️ 微小瑕疵

- 第 479 行和 481 行有重复的 `<!-- 鼠标拖尾粒子 -->` 注释
- 不影响功能，删一个即可
