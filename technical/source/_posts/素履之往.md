## 购入mac
由于想要一个能够带去上课的轻薄本，我开始找寻我的目标，在intel不太成熟的LunaLake和Mac生态的成熟的ARM架构之间，我选择了Mac，也是满足我想尝试新系统的愿望。我对性能没有很高要求，macbook air无疑是性价比之选，当然它在低价位给出的超大统一内存也是一个加分项，（虽然M1pro的无头骑士更香）算是一份意外之喜。在轻薄本的预算范围之内，Macbook air无疑满足了轻度文本编辑的需要同时，满足了Ai本地化部署的条件。因此在2026：这个金价与内存条齐飞，配置共钱包并缩的今天，Macbook成为了3开到4开区间的极致性价比之选————在2026花同样的钱买着2024年的显卡，配置还不如2024年，好命苦————此时我已什么都不缺。
OS：拿钱买外设太浪费了，因此现在用着当时次顶配的笔电，却有着最丐的外设，有线设备占满了笔记本的插口————外接显示器、电竞椅、拓展坞...都随他去吧，买一个Macbook应该算是很任性了，所以内存我可要好好地用推理进程占满你了啊喂！！！
## 搭建环境，捣鼓命令行
最开始的时候我为了快速地从github上拉取仓库，先通过共享文件夹把魔法配置好，甚至还手搓http网址（其实当时没有想到好办法，下载社交媒体又太冗余了，不符合我的初衷）。然后根据指引在本地设计好文件夹，然后开始配置环境。
```bash
xcode-select --install
#运行后会弹窗提示安装，Git 随 Command Line Tools 一起装好。不装完整 Xcode（几十 GB），只装命令行工具（几百 MB）
#将git指令添加到path里面（这点是比windows方便的）
export PATH="/usr/local/bin:$PATH"
#然后直接clone下ollama
git clone https://github.com/ollama/ollama.git
```
但是这个时候，我正准备下载Homebrew的时候，网络实在不行，即使使用了代理也不行（小珍珠都要掉出来了 mewm~ QAQ），配置清华的镜像源也不行（手搓http真的很难吧，啊喂！！！）。一夜过去了，我只能回到最初的办法，使用共享文件夹的方法，将我在windows上下好的dmg文件和zip压缩包一起打包过去（好像给植物人喂饭，该死の...）。当时打算打包了一个mac-setup的文件夹，但是不知道由于什么申必力量（有可能是读写权限的问题），导致我在mac中无法将共享文件夹拖拽至本地，故此我另寻他法。
于是我就开始考虑使用共享局域网下，在windows的主机上起了一个http服务
```bash
python -m http.server 8000
#在管理员模式下输入：
New-NetFirewallRule -DisplayName "Python HTTP 8000" -Direction Inbound -Protocol TCP -LocalPort 8000 -Action 
```
外面的人访问不到，因为windows的防火墙默认拦截入站流量，执行这条命令后，防火墙就会放行 8000 端口的入站请求。局域网内的其他人可以用http://我的ip:8000 访问我电脑上的服务了。
然后我就着手本地化部署了，下面是我参照的大肥鱼给出的文档[mac-setup](/attachments/素履之往.blog/mac-安装指南.txt)，但是实际上，我的内存在打开ollama的模型和docker的同时一瞬间就爆了，我的触控板突然断触，活动监视器爆红，swap内存飙升到200MB，给我人看傻了。具体原因是因为我想要运行一个openwebui窗口，这个项目在windows上看只有几十MB但是实际上，带上docker的虚拟环境总共就有7GB，这是我始料未及的（我之前使用虚拟机运行linux给了我虚拟机很轻巧的错觉，实际上windows太笨重了，根本不适合在macos系统上运行）。实际上我已经适应了没有ui的终端运行环境，加上macos简洁的文件管理方式，其实还挺轻便。以下是成功运行图[mac-desktop](/attachments/素履之往.blog/1.jpg)。
